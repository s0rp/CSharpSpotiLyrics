# Contributing to CSharpSpotiLyrics

First off, thank you for considering contributing to CSharpSpotiLyrics! Your interest is appreciated.

While this project was primarily developed by me, contributions that improve the tool and adhere to the guidelines below are welcome. Please remember the disclaimer noted in the README: this tool likely operates in a grey area regarding Spotify's Terms of Service, and contributors should be aware of the potential risks.

## How Can I Contribute?

There are several ways you can contribute:

### 1. Reporting Bugs

*   **Check Existing Issues:** Before submitting a new bug report, please search the existing [GitHub Issues](https://github.com/s0rp/CSharpSpotiLyrics/issues) (replace with your actual repo link) to see if the bug has already been reported.
*   **Provide Details:** If the bug hasn't been reported, create a new issue and provide as much detail as possible:
    *   Steps to reproduce the bug.
    *   What you expected to happen.
    *   What actually happened (include error messages or console output).
    *   Your environment (Operating System, .NET version).
    *   The specific command or input you used.

### 2. Suggesting Enhancements or Features

*   **Check Existing Issues/Requests:** Search the [GitHub Issues](https://github.com/s0rp/CSharpSpotiLyrics/issues) to see if a similar enhancement has already been suggested.
*   **Explain Clearly:** If it's a new idea, create a new issue detailing the enhancement:
    *   Describe the feature clearly.
    *   Explain why you think it would be a valuable addition.
    *   (Optional) Suggest how it might be implemented.

### 3. Code Contributions (Pull Requests)

If you'd like to contribute code to fix a bug or implement a feature:

1.  **Fork the Repository:** Create your own fork of the repository on GitHub.
2.  **Create a Branch:** Create a new branch in your fork for your changes (e.g., `git checkout -b feature/add-cool-feature` or `fix/resolve-login-bug`). Please use a descriptive branch name.
3.  **Make Your Changes:** Write your code. Try to adhere to the existing code style and C#/.NET conventions used in the project.
4.  **Build and Test:** Ensure your code builds successfully (`dotnet build`). If possible, test your changes thoroughly to ensure they work as expected and don't introduce new issues.
5.  **Commit Your Changes:** Use clear and concise commit messages explaining the "what" and "why" of your changes.
6.  **Push to Your Fork:** Push your branch to your forked repository on GitHub.
7.  **Open a Pull Request (PR):**
    *   Navigate to the main repository and open a Pull Request from your branch to the main repository's `main` (or appropriate default) branch.
    *   Provide a clear title and description for your PR. Explain the purpose of the changes and reference any related issues (e.g., "Fixes #123").
    *   I will review the PR. Be prepared to respond to feedback or make further changes if requested.

## Development Setup

1.  Ensure you have the .NET SDK installed (see Prerequisites in README).
2.  Clone your fork: `git clone https://github.com/s0rp/CSharpSpotiLyrics.git`
3.  Navigate to the project directory: `cd CSharpSpotiLyrics`
4.  Build the project: `dotnet build`
5.  Run the project: `dotnet run -- [arguments]` or `CSharpSpotiLyrics.exe [arguments]`

## Code Style

Please try to match the existing code style, formatting, and naming conventions found in the project. Generally, follow standard C# and .NET coding guidelines.

## Questions?

If you have questions about contributing, feel free to open an issue on GitHub.

Thank you again for your interest in CSharpSpotiLyrics!
